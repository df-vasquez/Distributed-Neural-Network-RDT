1. DESCRIPCIÓN GENERAL

Se implementa un protocolo de comunicación confiable sobre UDP denominado RDT-UDP,
orientado a soportar el intercambio de datos en un sistema de entrenamiento
distribuido de una red neuronal bajo arquitectura tipo Parameter Server.

El protocolo añade confiabilidad a UDP mediante control de errores, orden,
confirmación de recepción y retransmisión de paquetes.

---------------------------------------------------------------------

2. ARQUITECTURA DEL SISTEMA

El sistema está compuesto por:

- Un nodo maestro encargado de la coordinación del entrenamiento.
- Tres nodos esclavos encargados del cómputo local.

El nodo maestro distribuye datos y parámetros, y consolida los gradientes
producidos por los nodos esclavos.

---------------------------------------------------------------------

3. FORMATO DEL DATAGRAMA

Cada paquete tiene un tamaño fijo de 500 bytes con la siguiente estructura:

- FLAGS (2 bytes): tipo de paquete.
- SEQ (4 bytes): número de secuencia.
- ACK/NACK (4 bytes): confirmación o solicitud de retransmisión.
- PAYLOAD (489 bytes): datos de la aplicación.
- CHECKSUM (1 byte): verificación de integridad.

---------------------------------------------------------------------

4. DESCRIPCIÓN DE CAMPOS

FLAGS:
Indica el tipo de paquete:
- Inicio de transmisión
- Paquete intermedio
- Fin de transmisión
- Paquete de control (ACK/NACK)

SEQ:
Número de secuencia del paquete para control de orden y detección de pérdidas.

ACK/NACK:
Indica el siguiente paquete esperado o solicita retransmisión de un paquete
específico en caso de error.

PAYLOAD:
Contiene los datos de la aplicación (dataset, parámetros o gradientes).
Se rellena con padding cuando es necesario para mantener el tamaño fijo.

CHECKSUM:
Valor de verificación calculado como la suma de los bytes del paquete módulo 256.
Permite detectar corrupción de datos durante la transmisión.

---------------------------------------------------------------------

5. MECANISMOS DE CONFIABILIDAD

El protocolo implementa los siguientes mecanismos:

- Detección de errores mediante checksum.
- Control de orden mediante números de secuencia.
- Confirmación de recepción mediante ACK.
- Detección de pérdidas mediante NACK.
- Retransmisión mediante temporizador (timeout con select).
- Estimación de RTT basada en el algoritmo de Karn.
- Control de congestión mediante backoff exponencial.

---------------------------------------------------------------------

6. APLICACIÓN AL ENTRENAMIENTO DISTRIBUIDO

El protocolo se utiliza para:

- Distribuir el dataset entre el nodo maestro y los nodos esclavos.
- Enviar parámetros iniciales del modelo.
- Transmitir gradientes desde los esclavos hacia el maestro.
- Garantizar sincronización en el entrenamiento distribuido.

El sistema realiza una única época de entrenamiento con agregación centralizada
de gradientes en el nodo maestro.
